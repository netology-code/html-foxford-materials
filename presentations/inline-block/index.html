Что такое строчно-блочные элементы

 

Строчно-блочные элементы (inline-block) — это элементы, объединяющие в себе свойства блочных элементов (display: block) и строчных элементов (display: inline).

Главное отличие от блочных элементов

Строчно-блочные элементы не добавляют перенос строки после себя, следовательно элементы могут следовать друг за другом горизонтально.

Главное отличие от инлайновых элементов

Строчно-блочные элементы позволяют задавать для себя ширину и высоту, а так же верхние и нижние внешние отступы (margin-top, margin-bottom).

Основные свойства строчно-блочных элементов

  • Как блочный элемент, имеет ширину и высоту, а также форму прямоугольника;
  • Как у инлайнового элемента, ширина и высота определяется содержимым;
  • Как у блочного элемента, ширину и высоту можно поменять, можно задать margin-top/margin-bottom;
  • Как инлайновый элемент, НЕ переносит следующий элемент на новую строку.

Свойство display

С помощью свойства display мы можем задать любое отображение — сделать элемент блочным, строчным или строчно-блочным. Но сегодня нас интересует значение inline-block, которое отвечает как раз за строчно-блочные или, как еще говорят, инлайн-блочные элементы.

      
        ul li {
          display: inline-block;
        }

        .my-element {
          display: inline-block;
        }

Селекторы классов

Для стилизации кроме селекторов тегов (div, span, ul, li, p и т.д.) мы можем применять селекторы классов. С помощью селектора класса мы можем «выбрать» нужный нам элемент по значению атрибута class.

      
        .list {
          padding-left: 0;
        }

        .list-item {
          list-style-type: none;
        }

Значение классов в современной верстке

На сегодняшний день, селекторы классов имеют огромное значение в верстке и наиболее употребимы. С помощью классов мы можем легко выбрать нужный нам элемент. Классы помогают избегать излишнего повторения кода в стилях.

II. Применение строчно-блочных элементов

Стилизация элементов

Мы можем задавать для элемента display: inline-block, чтобы обеспечить возможность его стилизации, а именно задавать для него ширину и высоту. Важно помнить, что изначально ширина строчно-блочных элементов зависит от контента!

Типичный пример — оформление ссылки в виде кнопки.

Делаем кнопку из ссылки

Частая практика задавать для кнопок min-width, чтобы кнопка не стала меньше определенного размера. С display: inline-block это можно сделать

      
        .link {
          display: inline-block;
          text-align: center;
          text-decoration: none;
          color: #fff;
          background-color: tomato;
          padding: 15px 20px;
          min-width: 100px;
        }

Горизонтальное расположение элементов

Мы можем изменять стандартное направление элементов (вертикально, сверху вниз), на горизонтальное.

Когда это нужно?

  • горизонтальное меню
  • карточки

III. Решение некоторых задач с помощью inline-block

Задача №1

Дано: список филиалов в виде списка, каждый элемент списка содержит название города, которое является ссылкой

Цель: города должны располагаться горизонтально, у каждой ссылки должен быть задан фон, расстояние между ссылками должно составлять 5px

Решение

Приведена только часть кода. Полный код в живом примере

      
        .list {
          font-size: 0;
        }

        .list-item {
          display: inline-block;
          list-style-type: none;
          margin: 0 5px 5px 0;
        }

        .list-link {
          display: inline-block;
          padding: 10px 20px;
        }
Живой пример

Задача №2

Дано: блок «Преимущества нашей компании»; каждое преимущество состоит из цифры, заголовка и описания

Цель: расположить преимущества в строку и центрировать их

Решение

Приведена только часть кода. Полный код в живом примере

      
        .advantages-list {
          text-align: center;
        }

        .advantages-item {
          display: inline-block;
          width: 33.333%;
          margin-right: -4px;
        }
Живой пример

Обратили внимание на font-size: 0 в первом примере и отрицательный margin-right во второй задаче?

IV. Проблема при использовании inline-block

Описание проблемы

У строчно-блочных элементов появляется отступ справа, который мы не задавали. Это не дает нам просто задать ширину элементу в 50% для 2-х колонок, 33.333% для 3-х колонок и т.д.

Причина

Этот отступ достался от строчных элементов. Т.к. строчные элементы должны встраиваться в текст между другими словами, то появляется отступ, равный межсловному интервалу.

Живой пример

Решение

    Решить эту проблему можно один из следующих способов:
  • font-size: 0 для родителя
  • отрицательный margin-right
  • отрицательный word-spacing для родителя
Живой пример

font-size: 0

Межсловное расстояние зависит от размера шрифта, чем он меньше, тем меньше расстояние между словами. Соответственно расстояние равно нулю при нулевом размере шрифта.

Минус: Свойство font-size наследуется. Для вложенных элементов нужно задавать размер шрифта.

Отрицательный margin-right

Можно «притянуть» соседний элемент с помощью отрицательно margin-right.

Минус: для каждого шрифта и под каждый размер значение margin-right нужно подбирать отдельно.

Отрицательный word-spacing для родителя

Можно задать отрицательно межсловное расстояние с помощью свойства word-spacing.

Минус: для каждого шрифта и под каждый размер значение word-spacing нужно подбирать отдельно.

Дополнительно

Кого-то может удивить, что иногда не работает свойство text-align: justify. Например, мы хотим выровнять элементы по всей ширине родительского контейнера. Если элементов меньше, чем нужно для переноса на другую строку, то text-align: justify не будет работать. Это поведение точно такое же, как и при выравнивании текста. Если вы выравниваете текст по ширине, то несколько слов в последней строке должны остаться выровненными по левому краю.

Решение

Это решается с помощью добавления псевдоэлемента. Возможно, это сейчас сложно для понимание, но в будущем вам может пригодиться это решение.
